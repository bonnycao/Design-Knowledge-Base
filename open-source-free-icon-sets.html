How to use icons in development (CDN examples)

Most icon libraries support multiple delivery methods (SVG files, npm packages, webfonts). For simple prototyping or static sites, CDN is easiest. For production apps, bundling icons (tree-shaking) is often better for performance.

1) CSS / Webfont icon sets (fastest setup)

Remix, Tabler webfont, Phosphor webfont variants, Font Awesome free.

<!-- Example: Remix Icon -->
<link rel="stylesheet" href="https://cdn.jsdelivr.net/npm/remixicon@4.6.0/fonts/remixicon.css">

<i class="ri-home-line"></i>
    
<!-- Example: Font Awesome Free -->
<link rel="stylesheet" href="https://cdnjs.cloudflare.com/ajax/libs/font-awesome/6.5.0/css/all.min.css">

<i class="fa-solid fa-user"></i>

2) JS replace() icon sets (inline SVG via script)

Feather and Lucide both support replacing placeholders with SVG.

<!-- Example: Feather -->
<script src="https://unpkg.com/feather-icons@4.29.2/dist/feather.min.js"></script>

<i data-feather="camera"></i>

<script>
  feather.replace();
</script>
    
<!-- Example: Lucide (UMD) -->
<script src="https://unpkg.com/lucide@latest/dist/umd/lucide.min.js"></script>

<i data-lucide="alarm-clock"></i>

<script>
  lucide.createIcons();
</script>

3) Web components (Ionicons)

<script type="module" src="https://unpkg.com/ionicons@7.4.0/dist/ionicons/ionicons.esm.js"></script>
<script nomodule src="https://unpkg.com/ionicons@7.4.0/dist/ionicons/ionicons.js"></script>

<ion-icon name="home-outline"></ion-icon>

4) Google Material Symbols (Google Fonts)

<link rel="stylesheet" href="https://fonts.googleapis.com/css2?family=Material+Symbols+Outlined" />

<span class="material-symbols-outlined">home</span>
